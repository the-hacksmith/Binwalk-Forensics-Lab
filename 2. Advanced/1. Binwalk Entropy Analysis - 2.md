# 📘 Manual Entropy Analysis (Without Graph)
## 🧩 What is happening here?

Sometimes when you run:
```
binwalk -E file
```
you may not see the graph because of system errors (like your Numba error).

👉 But as a SOC Analyst, you should not stop.
Instead, you switch to manual analysis to check if data is normal or suspicious.


## 🧠 Core Idea

Inside every file:
- Some parts are normal (low entropy) → readable, structured
- Some parts are hidden/compressed/encrypted (high entropy) → random

👉 Your job is to find where the data becomes random


## ⚙️ Step 1: Start with Binwalk (Find Suspicious Offset)
```
binwalk infected.jpg
```
👉 Example output:

102400 → ZIP archive

✔️ This tells you:

“There is something hidden starting at this position (offset)”


## ⚙️ Step 2: Check Readable Content
```
strings infected.jpg | less
```
🧠 What you are doing:
You are checking:
👉 “Can I read anything inside this file?”

✔️ What you will see:
- Normal part → readable text
- Hidden part → no readable text

👉 If text suddenly disappears → that area is suspicious


## ⚙️ Step 3: Look at Raw Data (Deep Inspection)
```
hexdump -C infected.jpg | less
```
🧠 What you are doing:

You are looking at raw binary data

✔️ How to understand:
🔹 Normal (Low Entropy)
- Repeating patterns
- Some readable text
🔹 Suspicious (High Entropy)
- Random characters
- No clear pattern
- Looks “messy”

👉 That messy area = possible hidden data


## ⚙️ Step 4: Focus on Suspicious Area (IMPORTANT)

Use the offset you saw earlier:
```
dd if=infected.jpg bs=1 skip=102400 count=200 | hexdump -C
```
🧠 What is happening:
- skip=102400 → go directly to suspicious part
- count=200 → read small portion

👉 You are zooming into the suspicious area 🔍


## ⚙️ Step 5: Confirm What is Inside
```
dd if=infected.jpg bs=1 skip=102400 count=1000 > part.bin
file part.bin
```
🧠 What is happening:

You extract that part and ask:
👉 “What type of file is this really?”

✔️ Output may show:
- ZIP archive
- Executable
- Unknown data


## 🧠 What You Actually Did (Important Understanding)

You:
- Found suspicious location (Binwalk)
- Checked if data is readable (strings)
- Inspected raw structure (hexdump)
- Zoomed into exact region (dd)
- Confirmed file type (file)

👉 This is real forensic analysis process


## 💡 Key Understanding
- Hidden data looks random (high entropy)
- Normal data looks structured (low entropy)
- Even without graph, you can still detect suspicious regions
- Tools don’t matter — your thinking matters more


## 📝 Final Conclusion
In this exercise, I learned how to manually analyze a file to detect hidden or suspicious data without relying on graphical output. By using tools like strings, hexdump, and dd, I was able to identify random (high entropy) regions and confirm the presence of hidden data.


## 🧠 Memory Tip
👉 “If data looks random, something is hidden there.”
