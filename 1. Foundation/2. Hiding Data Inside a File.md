# 📘 Title: Detecting Hidden Data

## 🧩 Objective
To simulate how attackers hide data inside files and how Binwalk can detect it.


## ⚙️ Steps Performed
### 1. Create a Secret File
```
echo "This is a secret malware simulation" > secret.txt
```
✔️ Created a text file with hidden content

### 2. Compress the File
```
zip secret.zip secret.txt
```
✔️ Converted the file into a ZIP archive


### 3. Hide ZIP inside Image
```
cat test.jpg secret.zip > infected.jpg
```
✔️ Combined image and ZIP into one file (data hiding technique)


### 4. Scan with Binwalk
```
binwalk infected.jpg
```
✔️ Detected both:
- JPEG image
- Hidden ZIP file


### 5. Extract Hidden Data
```
binwalk -e infected.jpg
```
✔️ Extracted hidden files from the image


## 🧠 What I Learned
- Attackers can hide malicious data inside normal files
- File extensions do not always reflect real content
- Binwalk can detect embedded files using signatures
- Extraction helps recover hidden payloads


## 💡 Key Concept
File hiding (simple steganography) can be detected using Binwalk


## 🔑 Key Commands
```
binwalk infected.jpg
binwalk -e infected.jpg
```


## 📝 Conclusion
In this exercise, I simulated a real attack scenario by hiding a ZIP file inside an image. Using Binwalk, I successfully detected and extracted the hidden data, demonstrating how SOC Analysts investigate suspicious files.
