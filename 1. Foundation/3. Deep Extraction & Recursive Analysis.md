# 📘 Title: Deep Extraction & Recursive Analysis
## 🧩 Objective
To perform deep analysis on a suspicious file and extract all hidden data layers using Binwalk recursive mode.


## ⚙️ Step Performed
### 1. Run Deep Extraction Command
```
binwalk -Me infected.jpg
```
✔️ This command performs both extraction and recursive scanning.


### 🧠 Detailed Explanation
- The -e option tells Binwalk to extract detected files
- The -M option enables recursive analysis

When combined (-Me), Binwalk:
- Scans the main file
- Extracts detected embedded files
- Scans the extracted files again
- Repeats the process until no more hidden data is found
👉 This process is called recursive extraction


## 🔍 Output Analysis
After running the command, a folder is created:
```
_infected.jpg.extracted/
```

Inside this folder, you may find:
- Extracted ZIP archive
- Extracted text file (secret.txt)


## 🧠 What I Learned
- Some files contain multiple layers of hidden data
- Simple scanning may not reveal all hidden content
- Recursive analysis helps uncover deeply hidden payloads
- Attackers use layered hiding techniques to evade detection
- Binwalk can fully extract all layers automatically


## 💡 Key Concept
Recursive Extraction = analyzing files layer by layer until the deepest content is revealed


## 🔑 Key Command
```binwalk -Me <filename>
```


## 📝 Conclusion
In this exercise, I used Binwalk recursive mode to deeply analyze a suspicious file. I was able to extract all embedded layers, demonstrating how hidden data can exist in multiple levels and how forensic tools can uncover them completely.
